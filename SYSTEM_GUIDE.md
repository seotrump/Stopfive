# StopFive 시스템 구조 및 부분 수정 가이드

이 문서는 StopFive 서비스의 전체적인 아키텍처와 주요 소스 코드의 위치를 설명하기 위해 작성되었습니다. 만약 개발 쿼터(제한)가 소진되어 다른 AI 언어 모델(Claude, ChatGPT 등)을 통해 수동으로 코드를 수정해야 할 때, 이 가이드를 참고하여 필요한 부분만 발췌하여 질문하면 훨씬 빠르고 정확한 답변을 얻을 수 있습니다.

---

## 1. 사용 기술 스택
- **프레임워크:** Next.js (React 기반)
- **스타일링:** Tailwind CSS
- **데이터베이스 & 인증:** Supabase (PostgreSQL 기반 BaaS)
- **언어:** TypeScript (JavaScript의 타입 확장 버전)

---

## 2. 주요 파일 및 디렉토리 구조

서비스의 거의 모든 주요 로직은 크게 두 개의 파일에 집중되어 있습니다. 이 두 파일만 이해하면 시스템의 90% 이상을 수정할 수 있습니다.

### 📍 프론트엔드 UI 및 상태 관리 (가장 자주 수정하게 될 파일)
**`src/app/page.tsx`**
이 파일은 웹사이트의 모든 화면(홈 화면, 로그인 화면, 유저 대시보드, 관리자 대시보드)을 그리는 핵심 파일입니다. 단일 페이지 애플리케이션(SPA) 방식으로, 이 파일 하나에서 상태값(`state`)에 따라 화면을 갈아 끼우는 형태로 작동합니다.

* **화면을 결정하는 핵심 변수 (State):**
  * `authMode`: `'home'`, `'login'`, `'register'`, `'dashboard'` 네 가지 상태를 가집니다. 사용자가 로그인하기 전 화면을 제어합니다.
  * `userTab`: 유저 대시보드 내에서의 메뉴 상태 (`inbox`, `sent`, `settings` 등)
  * `adminTab`: 관리자 대시보드 내에서의 메뉴 상태

* **수정 팁:**
  * 화면 디자인(버튼 색상, 여백, 텍스트)을 바꾸고 싶다면 이 파일의 HTML(`<div>`, `<button>` 등) 및 `className="tailwind-클래스"` 부분을 수정해야 합니다.
  * 새로운 메뉴나 팝업 모달을 추가하고 싶다면 이 파일에 새로운 상태값(`useState`)을 추가하고 조건부 렌더링을 구현하면 됩니다.

### 📍 백엔드 로직 및 데이터베이스 통신
**`src/lib/db.ts`**
이 파일은 Supabase 데이터베이스와 통신하여 유저 정보를 가져오거나 메일을 보내고 저장하는 역할을 합니다.

* **주요 함수:**
  * `getCurrentUser()`: 현재 로그인된 유저 세션 가져오기
  * `loginUser()`, `registerUser()`: 로그인 및 회원가입 처리
  * `getLocalEmails()`, `sendReply()`: 메일 목록 불러오기, 답장 전송하기
* **수정 팁:**
  * 회원가입 시 추가 정보를 받고 싶거나, 메일을 보낼 때 처리되는 로직을 변경하고 싶다면 이 파일을 수정해야 합니다.
  * 데이터베이스(Supabase) 테이블 구조가 바뀌면 이곳의 함수들도 맞게 수정해야 합니다.

## 3. 타 언어 모델(ChatGPT, Claude 등)을 활용한 부분 수정 가이드

AI 모델의 답변 제한에 걸리거나 직접 코드를 수정하고 싶을 때는, **전체 코드를 복사해서 붙여넣지 마시고(코드가 너무 길어 AI가 혼란을 겪거나 거부할 수 있습니다), 필요한 부분만 잘라서 질문하는 것이 핵심**입니다.

### 상황별 프롬프트(질문) 작성 예시

#### 👉 상황 A: 화면의 특정 UI(디자인/텍스트)를 바꾸고 싶을 때
1. `src/app/page.tsx`를 에디터로 엽니다.
2. `Ctrl + F` (또는 `Cmd + F`)를 눌러 바꾸고자 하는 텍스트(예: "내 대시보드로 돌아가기")를 검색합니다.
3. 해당 코드가 포함된 주변 블록(보통 `<div className="...">` 부터 `</div>` 까지)을 복사합니다.
4. AI에게 다음과 같이 질문합니다:
   > "Next.js와 Tailwind CSS를 사용하고 있어. 아래 코드에서 '내 대시보드로 돌아가기' 버튼의 색상을 파란색 계열로 바꾸고 아이콘을 하나 추가해줘. 코드는 다음과 같아: (복사한 코드 붙여넣기)"

#### 👉 상황 B: 기능 로직(버튼 클릭 시 일어나는 일)에 버그가 생겼을 때
1. `src/app/page.tsx` 파일 위쪽에 있는 `// 핸들러` 주석 부분이나 `const handle...` 로 시작하는 함수들을 찾습니다.
2. 오류가 발생한 함수(예: `handleLogin` 또는 `handleSendReply`) 전체를 복사합니다.
3. AI에게 다음과 같이 질문합니다:
   > "아래는 React 컴포넌트 내부의 로그인 핸들러 함수야. 새로고침 시 상태가 꼬이는 버그가 있어서, 로그인 직후 authMode가 무조건 'dashboard'로 고정되도록 코드를 수정해줘: (복사한 코드 붙여넣기)"

#### 👉 상황 C: 데이터베이스 처리 방식(메일 저장 등)을 바꾸고 싶을 때
1. `src/lib/db.ts` 파일을 엽니다.
2. 관련된 함수(예: `sendReply`)를 복사합니다.
3. AI에게 다음과 같이 질문합니다:
   > "Supabase를 사용 중인 TypeScript 환경이야. 아래 함수에서 메일을 답장할 때, 기존의 원래 메일을 '읽음' 처리 뿐만 아니라 'is_replied = true' 라는 컬럼도 함께 업데이트하도록 코드를 고쳐줘: (복사한 코드 붙여넣기)"

## 4. 로컬 환경에서 적용하고 테스트하기

다른 AI가 수정해준 코드를 로컬에 적용하는 방법은 다음과 같습니다.

1. AI가 알려준 코드를 기존 파일의 알맞은 위치에 덮어쓰기 합니다.
2. 터미널(명령 프롬프트)에서 해당 프로젝트 폴더로 이동한 뒤, 아래 명령어를 입력하여 컴파일 에러가 없는지 먼저 확인합니다.
   ```bash
   npm run build
   ```
3. 에러가 나지 않는다면 로컬 개발 서버를 켜서 직접 화면을 확인합니다.
   ```bash
   npm run dev
   ```
4. 모든 것이 정상 작동한다면 아래 명령어로 깃허브에 백업(푸시)합니다.
   ```bash
   git add .
   git commit -m "원하는 수정 내역 설명"
   git push
   ```
